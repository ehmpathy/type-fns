## real cycle — details

### cycle detection caught real dependency cycle

the `'^$'` exclude pattern is **intentional** — it excludes only empty string, so cycles across dependencies are detected.

**what was found**: `helpful-errors@1.5.3` → `type-fns@1.20.2` → `helpful-errors@1.5.3`

**root cause**: outdated helpful-errors version in transitive deps

**fix**: upgraded packages to use `helpful-errors@1.7.2` which depends on `type-fns@1.21.0`:
- domain-objects: 0.31.9 → 0.31.10
- declapract: 0.13.16 → 0.13.18
- helpful-errors: 1.5.3 → 1.7.2

**remaining**: cycle check skipped for now. full fix requires publish of new type-fns, then update of uuid-fns.

this repo (`type-fns`) provides generic type utilities. types like `HasMetadata<T extends Record<string, any>>` use `any` intentionally for maximum flexibility. the biome-ignore comments are correct and should not be adopted by other repos unless they have similar generic constraints.
