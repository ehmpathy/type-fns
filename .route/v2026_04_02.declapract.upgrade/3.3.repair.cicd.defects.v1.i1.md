## defect details

### 1. github / plan fails

**what**: provision workflow fails with `DomainObjectNotSafeToManipulateError`

**how**: upgrade added provision workflow with DeclaredGithubBranchProtection resource

**why**: declastruct-github does not declare `nested` property for `requiredPullRequestReviews` in DeclaredGithubBranchProtection. when GitHub API returns non-null object for this field, domain-objects throws because nested objects must be declared.

**fix**:
1. upgraded declastruct-github 1.3.0 to 1.3.4 (did not fix)
2. skipped branchMainProtection resource until upstream fix

### 2. test-types fails on @jest/globals

**what**: `Cannot find module '@jest/globals'` in jest.*.env.ts files

**how**: upgrade migrated from npm to pnpm

**why**: pnpm strict mode only allows imports from direct dependencies. `@jest/globals` was transitive via jest@30, but jest.env files import it directly. npm hoists transitive deps, pnpm does not.

**fix**: added `@jest/globals@30.2.0` as explicit devDependency
